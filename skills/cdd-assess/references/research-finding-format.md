# Research Finding Format

Use this format to persist reusable regulatory grounding. Research findings should be written after reading primary or high-trust sources, not from model recall.

File name:

```text
source-research/0001-short-topic.md
```

Template:

```md
# {Requirement Or Principle}

**Jurisdiction:** {country / region}
**Regulator / authority:** {name}
**Source:** {official title and URL}
**Source date:** {publication or revision date, if available}
**Accessed:** YYYY-MM-DD
**Applies to:** {client type / activity / report section}

## Requirement

{What the source requires. Use concise paraphrase and short compliant quotes only when needed.}

## Interpretation

{How the requirement should be understood for onboarding assessment work.}

## Rationale / Principle

{Why the requirement exists: risk, control objective, customer harm, AML/CFT concern, market integrity, etc.}

## Report Implications

- {What factual records should be captured}
- {What derivations or evidence checks may be needed}
- {What observations can be made if evidence is missing, conflicting, or weak}

## Limits And Uncertainty

{Jurisdiction limits, source freshness concerns, unresolved interpretive issues, or areas needing human legal/compliance judgment.}
```

## Source Rules

- Prefer official regulator, statute, FIU, government, court, or supervisory material.
- Use recognized standards bodies when they are the relevant authority or accepted benchmark.
- Use secondary sources only to find primary sources or understand terminology.
- If a source cannot be verified, record the gap and do not use it as final authority.
